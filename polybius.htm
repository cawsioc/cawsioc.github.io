How it works

A Polybius Square arranges a keyword or phrase as the start of the ciphertext alphabet and appends any characters that didn't occur in the keyword on to the end of that alphabet exactly as a standard keyword cipher does. However, the letters "i" and "j" are then collapsed into the same symbol ("i" in this case), to produce a ciphertext alphabet of 25 characters, which is then placed in a 5 x 5 grid. To encrypt, the position of the plaintext character in the 5 x 5 grid is used to obtain a column number and row number. The character is then represented by the resulting 2-digit pair. To decrypt, the receiver constructs the grid with the same keyword, and reads off the plaintext character at the location indicated by the column and row elements of each pair of digits in the ciphertext. The fact that only a pair of small values is needed to represent each letter lends itself to disguising or representing them in a wide variety of ways.

Whilst the Polybius Square can be used as a cipher (although this was not it's original purpose) it is vulnerable to frequency analysis in the same way as all other monoalphabetic substitution ciphers.
